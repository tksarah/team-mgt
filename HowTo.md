# Team Progress Manager 使い方

このドキュメントは、Team Progress Manager `v1.3` の日常操作をまとめたものです。

一般的な IT スキルを持つ利用者を前提に、専門的なデータベース知識がなくても使えるよう、操作手順と注意点を中心に説明します。

## 1. はじめに

Team Progress Manager は、チームのタスク、担当者、進捗状況、依存関係をまとめて管理するためのアプリです。

主な機能は以下です。

- タスク一覧での登録・編集・確認
- 検索、カテゴリ、ステータスによる絞り込み
- 完了済みタスクのアーカイブ
- 依存マップによるタスク間の関係確認
- カテゴリ、担当者、ステータス、優先度のマスタ管理
- Excel 出力・取込
- 共有フォルダ上のデータファイルを使ったチーム運用

チームで使う場合は、全員が同じ「データファイル」を開きます。編集できるのは一度に 1 人だけです。他の人は同じデータを確認できますが、編集はできません。

このドキュメントでは、タスクやマスタの情報を保存している SQLite ファイルを「データファイル」と呼びます。

## 2. 起動する

ポータブル版を使う場合は、配布フォルダ内の `team-mgt.exe` をダブルクリックして起動します。

インストール作業が不要な構成の場合、`team-mgt.exe` と関連ファイルを同じフォルダに置いたまま利用してください。

初回起動時は、アプリが利用するデータファイルが自動的に用意されます。個人で試すだけであれば、そのまま利用できます。チームで共有する場合は、次の章の手順で共有フォルダ上のデータファイルを開きます。

`team-mgt.exe` を USB メモリや共有フォルダから直接起動する場合、PC やネットワーク環境によっては動作が遅くなることがあります。安定して利用するには、アプリ本体は各 PC に置き、データファイルだけを共有フォルダに置く運用を推奨します。

## 3. チーム用データを開く

チームで同じデータを使う場合は、共有フォルダ上にあるデータファイルを開きます。

1. 画面上部の「データファイル」欄を確認します。
2. 「開く」を押します。
3. 共有フォルダ上の `.sqlite` または `.db` ファイルを選択します。
4. 画面の表示が切り替わり、共有データのタスク一覧が表示されることを確認します。

チーム運用では、全員が同じファイルを選ぶことが重要です。似た名前のファイルや古いバックアップファイルを開かないよう、保存場所とファイル名をチーム内で決めておくと安全です。

まだ共有用のデータファイルがない場合は、共有フォルダ上に新しいデータファイルを作成してから、チーム全員でそのファイルを開く運用にします。フォルダだけを指定するのではなく、実際に利用するデータファイルを指定してください。

## 4. 編集を開始・終了する

タスクや設定を変更するには、先に編集を開始します。これは、複数人が同時に編集して内容がぶつかることを防ぐための仕組みです。

編集を開始する手順は以下です。

1. 右上の「利用者名」に自分の名前を入力します。
2. 「編集開始」を押します。
3. 表示が「編集可能」になったことを確認します。

「編集可能」になると、保存、削除、Excel 取込などの変更操作が行えます。

作業が終わったら「編集終了」を押します。アプリを閉じた場合も、自分が取得した編集状態は自動で解除されます。

他の利用者が編集している間は、データの確認はできますが、変更操作はできません。表示されている利用者名や端末名を確認し、必要に応じてチーム内で声をかけてください。

## 5. タスクを登録・編集する

「タスク一覧」タブでタスクを追加・編集します。

新しいタスクを登録する手順は以下です。

1. 「タスク一覧」タブを開きます。
2. 左側のタスク編集欄が閉じている場合は「開く」を押します。
3. 必要な項目を入力します。
4. 「保存」を押します。

主な入力項目は以下です。

| 項目 | 入力内容 |
| --- | --- |
| タスク名 | 作業内容がわかる名前を入力します。 |
| カテゴリ | 業務区分やプロジェクトなどを選びます。 |
| ステータス | 未着手、進行中、完了などの状態を選びます。 |
| 優先度 | 高・中・低など、作業の優先度を選びます。 |
| 開始日 | 作業を開始する日を入力します。 |
| 期日 | 作業の締切日を入力します。 |
| 担当者 | 主に対応する人を選びます。 |
| 依存タスク | 先に終わっている必要があるタスクがあれば指定します。 |
| メモ | 補足事項や連絡事項を入力します。 |

タスク名、カテゴリ、担当者、ステータス、優先度、開始日、期日は必須です。開始日は期日以前の日付にしてください。

依存タスクの注意点は以下です。

- 依存タスクは 1 件まで指定できます。
- 自分自身を依存先にすることはできません。
- A が B に依存し、B が A に依存するような循環関係は作れません。

## 6. タスクを確認・絞り込む

タスク一覧では、条件を指定して必要なタスクだけを表示できます。

- 検索キーワード: タスク名やメモなどから探すときに使います。
- カテゴリ: 特定の業務区分やプロジェクトだけを表示します。
- ステータス: 未着手、進行中、完了などで絞り込みます。
- アーカイブ表示: 通常一覧から隠した完了タスクを確認します。

既存タスクを編集する手順は以下です。

1. 一覧で対象タスクを探します。
2. 対象行の「編集」を押します。
3. 左側の編集欄に内容が読み込まれます。
4. 内容を修正し、「保存」を押します。

完了済みのタスクは「アーカイブ」を使うと、通常の一覧から非表示にできます。あとから確認したい場合は「アーカイブを表示」にチェックを入れます。必要に応じて「復元」で通常の一覧に戻せます。

## 7. 依存関係を確認する

「依存マップ」タブでは、タスク同士のつながりをツリー形式で確認できます。どの作業が終わらないと次の作業に進めないかを確認するために使います。

| 表示方法 | 内容 |
| --- | --- |
| 全体 | 依存関係のあるタスクを全体表示します。 |
| 選択中心 | 選んだタスクを起点に、その配下の関係だけを確認します。 |

依存マップには、アーカイブされていないタスクだけが表示されます。過去の完了タスクを確認したい場合は、タスク一覧でアーカイブ表示を利用してください。

## 8. マスタを管理する

「設定」タブでは、タスク入力で使う選択肢を管理できます。これらの選択肢をマスタと呼びます。

管理できるマスタは以下です。

- カテゴリ
- 担当者
- ステータス
- 優先度

マスタに設定できる内容は以下です。

- 名前: 画面に表示される名称です。
- 色: 一覧で見分けやすくするための色です。
- 並び順: 選択肢として表示される順番です。
- 有効・無効: 新規入力時に選べるかどうかを切り替えます。

使わなくなったカテゴリや担当者は、すぐ削除せず「無効」にする運用がおすすめです。無効にすると新規入力の選択肢には出ませんが、過去のタスクの記録としては残せます。

## 9. Excel に出力する

「Excel 出力」を押すと、現在のデータを Excel ファイルとして保存できます。

出力した Excel は、バックアップ、一覧確認、チーム内共有の補助資料として利用できます。

出力されるシートは以下です。

- タスク
- マスタ

Excel 出力は確認やバックアップのための補助機能です。日常運用での正本は、アプリで開いているデータファイルです。Excel だけを更新しても、アプリの内容は自動では変わりません。

## 10. Excel から取り込む

Excel からデータを取り込むには、編集を開始している必要があります。

取り込みは、基本的にこのアプリの「Excel 出力」で作成したファイルを編集してから行う運用を推奨します。

取り込み手順は以下です。

1. 「編集開始」を押し、編集可能な状態にします。
2. 「Excel 取込」を押します。
3. 取り込みたい Excel ファイルを選択します。
4. 画面上のメッセージを確認します。

取り込み時の動きは以下です。

- 既存のデータと対応する行は更新されます。
- 新しい行は新規データとして追加されます。
- カテゴリ、担当者、ステータス、優先度は、既存のマスタに紐づけられます。

入力内容に不備がある行は取り込めません。画面上のメッセージ欄に表示された内容を確認し、Excel 側の該当行を修正してから、もう一度取り込んでください。

## 11. 削除とアーカイブの使い分け

不要になったタスクを履歴として残したい場合は、削除ではなくアーカイブを使います。

| 操作 | 意味 |
| --- | --- |
| アーカイブ | 完了済みタスクを通常一覧から非表示にします。 |
| 復元 | アーカイブ済みタスクを通常一覧に戻します。 |
| 削除 | タスクまたはマスタをデータから削除します。 |

削除したデータはアプリ上から戻せません。削除前に、必要に応じて Excel 出力を行うか、データファイルのコピーを保存してください。

## 12. 困ったとき

通常は「編集終了」またはアプリ終了時に編集状態が解除されます。同じ PC でアプリが異常終了した場合は、次回起動時に古い編集状態が整理されます。

別の PC の利用者が編集中と表示される場合は、表示されている利用者名と端末名を確認し、実際に編集していないことをチーム内で確認してから対応してください。

保存や取り込みができない場合は、以下を確認してください。

- 編集可能な状態になっているか。
- 共有フォルダにアクセスできるか。
- 他の人が編集中でないか。
- Excel 取込の場合は、画面上のエラーメッセージに該当行や原因が表示されていないか。

## 13. データ管理の注意点

このアプリでは、タスクや設定の内容をデータファイルとして保存します。日常利用では中身の仕組みを意識する必要はありませんが、次の点に注意してください。

正本データについての考え方は以下です。

- 正本データは、アプリで開いているデータファイルです。
- Excel ファイルは、確認・バックアップ・移行のための補助ファイルです。
- チーム運用では、全員が同じ共有フォルダ上のデータファイルを開いてください。

やってはいけないことは以下です。

- データファイルを Excel やテキストエディタで直接開いて編集しないでください。
- 誰かが編集中のときに、共有フォルダ上のデータファイルを移動・削除・名前変更しないでください。
- バックアップファイルと本番用ファイルを同じ名前で混在させないでください。

おすすめのバックアップ運用は以下です。

- 作業前や大きな取り込み前に Excel 出力を行います。
- 必要に応じて、データファイル自体も日付付きでコピーして保管します。
- バックアップファイルは `backup` などのフォルダに分けて保存し、通常利用するファイルと混同しないようにします。

データファイルの拡張子は環境により `.sqlite` または `.db` の場合があります。一般利用者は拡張子を意識する必要はありませんが、共有フォルダ上で「チームが使う正式なファイル」を 1 つに決めて運用してください。
